# EncyHub 修复计划书 2.0 — 深度对比审查

> 基于原项目 AdbMaster、HaruRuntimeGMClient 的逐行对比，列出所有功能缺失与实现偏差。

## 一、ADB Master 问题清单

### 1.1 [P0] Logcat 按钮完全无响应

**现象**: 点击"开始"按钮无任何反应，无 press 状态反馈。

**根因分析**:
前端 `toggleLogcat()` 通过 WebSocket 连接到 `/api/adb_master/devices/{hw_id}/logcat`。该请求经过 Hub 的 WebSocket 代理 (`hub_core/api.py:155`) 转发到 AdbMaster 子进程。

问题链路:
1. Hub 代理使用 `websockets.connect(target_url)` 连接上游，但 AdbMaster 后端的 `websocket_logcat` 处理器 (`main.py:177`) 在 `await adb_mgr.start_logcat(dev.active_serial, on_line)` 处可能阻塞——如果 `start_logcat` 内部 await 了子进程的整个生命周期而非仅启动
2. `on_line` 回调中使用 `asyncio.create_task(ws.send_text(line))` 发送到 AdbMaster 内部的 WebSocket 连接池，但这些是 AdbMaster 自身的 WebSocket 对象，不是 Hub 代理的上游连接
3. 前端 `ws.onerror` 仅静默设置 `logcatRunning(false)`，无任何错误提示

**修复方案**:
- 检查 `adb_manager.py` 的 `start_logcat` 实现，确保它是非阻塞的（启动子进程任务后立即返回）
- 在 `websocket_logcat` 中，`start_logcat` 后的 `while True: receive_text()` 循环必须能正常执行
- 前端 `toggleLogcat` 添加错误反馈：`ws.onerror` 时 `toast.error('Logcat 连接失败')`
- 添加连接超时检测：如果 3 秒内未收到 `onopen`，提示用户

**涉及文件**:
- `tools/adb_master/adb_manager.py` — 检查 `start_logcat` 是否非阻塞
- `tools/adb_master/main.py:177-219` — WebSocket logcat 处理器
- `hub_core/api.py:155-215` — WebSocket 代理
- `frontend/src/pages/AdbMaster.jsx:136-167` — `toggleLogcat` 函数

### 1.2 [P0] 设备文件夹未创建

**原项目行为** (AdbMaster `path_utils.py:43-71`):
打开设备卡片时，自动创建:
```
Devices/{hardware_id}/
├── Local_Sync_Area/    ← 文件传输暂存区
└── logs/               ← Logcat 日志文件
```
- `hardware_id` 经过清洗：`:` → `_`，`.` → `_`
- Logcat 输出同时写入 `logs/logcat_YYYYMMDD_HHMMSS.log`

**EncyHub 现状**:
- 仅创建共享目录 `data/adb_master/temp/` 和 `data/adb_master/downloads/`
- 无设备专属目录
- Logcat 不保存到文件

**修复方案**:
- 后端：选择设备时（或首次操作时）创建 `data/adb_master/Devices/{safe_hw_id}/Local_Sync_Area/` 和 `logs/`
- Logcat 启动时同步写入日志文件到设备的 `logs/` 目录
- Push 操作的临时文件存放到设备的 `Local_Sync_Area/` 而非共享 temp
- Pull 操作的下载文件存放到设备的 `Local_Sync_Area/` 而非共享 downloads

**涉及文件**:
- `tools/adb_master/main.py` — 添加 `ensure_device_dirs()` 函数，在设备相关操作前调用
- `tools/adb_master/main.py:177-219` — Logcat 写文件逻辑

### 1.3 [P1] WiFi IP 地址未显示

```jsx
  <span className="text-xs text-[var(--sky)] font-mono">

  </span>
)}
```

**涉及文件**:

### 1.4 [P0] 拉取功能报错 + 本地路径不可填写

**现象**: `拉取失败: Unexpected token 'I', "Internal S"... is not valid JSON`

**根因分析**:
1. `handlePull` 在 `res.ok` 时调用 `res.blob()`，在 `!res.ok` 时调用 `res.json()`
2. 如果后端返回非 JSON 格式的错误（如 httpx 代理返回纯文本 500），`res.json()` 解析失败
3. Hub 代理 (`api.py:108-152`) 使用 `httpx.AsyncClient(timeout=30.0)` 转发请求，但 ADB pull 操作可能超过 30 秒（原项目超时 300 秒）
4. 后端返回 `FileResponse`，但 Hub 代理将其作为普通 HTTP 响应转发，可能丢失文件流

**原项目行为**:
- Push 和 Pull 都有 4 个可编辑路径输入框：
  - Push: 本地文件路径 + 远程目标路径
  - Pull: 远程文件路径 + 本地保存路径
- 所有路径都有历史记录下拉
- 本地路径可以手动输入任意路径

**EncyHub 现状**:
- Push: 只有文件选择器（浏览器 file input）+ 远程路径输入
- Pull: 只有远程路径输入，本地路径完全缺失
- Pull 下载方式是浏览器下载（blob URL），而非保存到用户指定路径

**修复方案**:
- 后端 Pull API 改为接受 `local_path` 参数，将文件拉取到指定本地路径，返回成功消息而非 FileResponse
- 如果未指定 `local_path`，默认保存到设备的 `Local_Sync_Area/`
- 前端添加本地保存路径输入框（带历史记录）
- 修复错误处理：`catch` 中先尝试 `res.text()` 再解析
- Hub 代理超时从 30s 增加到 300s（匹配原项目）

**涉及文件**:
- `tools/adb_master/main.py:259-278` — Pull API
- `hub_core/api.py:130` — 代理超时设置
- `frontend/src/pages/AdbMaster.jsx:196-225` — `handlePull` 函数
- `frontend/src/pages/AdbMaster.jsx:645-685` — Pull UI 区域

### 1.5 [P1] 路径历史按设备隔离

**原项目行为** (AdbMaster `config_manager.py:150-208`):
路径历史按 `hardware_id` + `path_type` 存储:
```json
{"devices": {"004adea1": {"history": {"push_local": [...], "push_remote": [...]}}}}
```

**EncyHub 现状**:
路径历史是全局的，不区分设备:
```json
{"path_history": {"push": [...], "pull": [...]}}
```

**修复方案**:
- 路径历史改为按设备存储：`config['devices'][hw_id]['history'][category]`
- API 改为 `GET /path-history/{hw_id}/{category}`
- 前端在切换设备时重新加载对应设备的历史

**涉及文件**:
- `tools/adb_master/config_manager.py` — 重构路径历史存储结构
- `tools/adb_master/main.py:336-355` — 路径历史 API
- `frontend/src/pages/AdbMaster.jsx:45-60` — `fetchPathHistory`

## 二、GM Console 问题清单

### 2.1 [P0] GM 按钮大小不一致

**原项目行为** (HaruRuntimeGMClient):
- 所有 GM 按钮固定高度 `h-24`（96px）
- 使用 `control-tile` 类统一尺寸
- 文本超长时截断（`line-clamp-2`），不撑大按钮

**EncyHub 现状**:
- Btn 类型：`p-3` 无固定高度，文本长度决定按钮大小
- SubBox 类型：同样无固定高度
- Toggle/Input 类型：高度由内容决定
- 结果：命令名称长短不一导致网格参差不齐

**修复方案**:
所有 GM 网格项统一添加固定高度和文本截断:
```jsx
// Btn 类型
className="p-3 h-24 ... overflow-hidden"
// 文本用 line-clamp-2 截断
<span className="line-clamp-2 text-sm">{node.name}</span>
```
- SubBox: 同样 `h-24`
- Toggle: `h-24 flex flex-col justify-between`
- Input: `h-24 flex flex-col justify-between`
- CustomGM 卡片: 同样 `h-24`

**涉及文件**:
- `frontend/src/pages/GmConsole.jsx:722-808` — GM 网格渲染
- `frontend/src/pages/GmConsole.jsx:838-875` — CustomGM 网格渲染

### 2.2 [P1] 自定义 GM 编辑按钮不可见

**现状分析**:
编辑按钮实际存在于代码中 (`GmConsole.jsx:851-862`)，使用 `opacity-0 group-hover:opacity-100` 仅在 hover 时显示。

**问题**:
- 触摸设备无法 hover
- 用户可能不知道 hover 才能看到编辑按钮
- 按钮太小（12px 图标 + 4px padding），难以点击

**修复方案**:
- 编辑/删除按钮改为始终可见（降低透明度而非完全隐藏）
- 或在卡片底部添加固定的操作栏
- 增大按钮点击区域

**涉及文件**:
- `frontend/src/pages/GmConsole.jsx:840-875` — CustomGM 卡片渲染

### 2.3 [P1] 缩放滑块范围问题

**原项目**: `min=2, max=8, step=1, default=5`

**EncyHub 现状**: `min=2, max=8`（与原项目一致）

**用户反馈**:
- 下限锁死在 2，用户希望能拉到 1
- 整体范围太窄，希望扩大

**修复方案**:
- 将范围改为 `min=1, max=10`
- 1 列模式适合查看长命令名称
- 9-10 列适合命令很多时的紧凑视图
- ZoomOut/ZoomIn 按钮的 `Math.max/Math.min` 同步更新

**涉及文件**:
- `frontend/src/pages/GmConsole.jsx:648-672` — 缩放控件
- `frontend/src/pages/GmConsole.jsx:651` — `Math.max(2, c - 1)` → `Math.max(1, c - 1)`
- `frontend/src/pages/GmConsole.jsx:666` — `Math.min(8, c + 1)` → `Math.min(10, c + 1)`

## 三、Hub 代理层问题

### 3.1 [P1] HTTP 代理超时过短

**现状**: `httpx.AsyncClient(timeout=30.0)` — 30 秒超时

**问题**: ADB 文件传输（push/pull/install）原项目超时 300 秒，大文件操作会被代理层提前截断。

**修复方案**:
- 将代理超时改为 `timeout=300.0`

**涉及文件**:
- `hub_core/api.py:130`

---

### 3.2 [P2] FileResponse 代理问题

**现状**: Pull API 返回 `FileResponse`，Hub 代理将整个文件内容读入内存后转发。

**问题**: 大文件会导致内存溢出。

**修复方案**:
- 使用 `httpx` 的流式响应 (`stream=True`) 转发文件内容
- 或改为 Pull API 返回本地路径信息而非文件流

**涉及文件**:
- `hub_core/api.py:108-152` — HTTP 代理
