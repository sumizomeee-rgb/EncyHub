## 二、详细分析

### 1.1 [P0] GM Console 刷新 404

**现象**: 在 `/gm_console` 页面按 F5 刷新，整个页面变成 `{"detail": "Not Found"}` JSON 文本。

**根因**: FastAPI 的 `StaticFiles(html=True)` 不是 SPA fallback。

当前挂载方式：
```python
# main.py:173
app.mount("/", StaticFiles(directory=str(FRONTEND_DIST), html=True), name="frontend")
```

`html=True` 的行为：
- `/` → 查找 `index.html` ✅
- `/gm_console` → 查找 `gm_console.html` 或 `gm_console/index.html` → 都不存在 → **404** ❌
- `/adb_master` → 同理 → **404** ❌

Vite 构建产物只有：
```
dist/
  index.html          ← SPA 入口
  assets/
    index-xxx.js      ← JS bundle
    index-xxx.css     ← CSS bundle
  favicon.svg
```

没有 `gm_console.html` 等文件，所以所有子路由刷新都会 404。

**修复方案**: 替换 StaticFiles 挂载为自定义 SPA fallback 路由：

```python
from fastapi.responses import FileResponse

# 挂载静态资源目录（JS/CSS/图片）
app.mount("/assets", StaticFiles(directory=str(FRONTEND_DIST / "assets")), name="assets")

# SPA fallback: 所有非 API 路径返回 index.html
@app.get("/{full_path:path}")
async def spa_fallback(full_path: str):
    # 先检查是否是真实文件（favicon.svg 等）
    file_path = FRONTEND_DIST / full_path
    if full_path and file_path.is_file():
        return FileResponse(file_path)
    return FileResponse(FRONTEND_DIST / "index.html")
```

路由匹配顺序保证 API 请求不受影响：
1. `hub_router` (`/api/hub/*`) → 优先匹配
2. `proxy_router` (`/api/{tool_id}/*`) → 优先匹配
3. `spa_fallback` (`/{full_path:path}`) → 兜底，返回 `index.html`

**涉及文件**: `main.py`

### 1.2 [P0] GM Console 游戏连接但 Web 无显示

**现象**: 游戏客户端 TCP 连接成功，控制台有 `[LUA: [RuntimeGM] GM Config Loaded.]` 日志，但 Web 端客户端列表为空。

**根因**: `on_client_data_update` 回调未设置。

事件链分析：

```
游戏客户端 TCP 连接
  ↓
_handle_client() → 创建 Client 对象 → on_update() 广播 → ✅ 前端收到 clients（但 gm_tree 为空）
  ↓
客户端发送 HELLO 包 → on_update() 广播 → ✅ 前端收到 device/platform 信息
  ↓
客户端发送 GM_LIST 包 → c.gm_tree = data → on_client_data_update(cid)
  ↓
❌ on_client_data_update 为 None → 不广播 → 前端永远看不到 gm_tree
```

`server_mgr.py:170-173`:
```python
elif t == "GM_LIST":
    c.gm_tree = pkt.get("data", [])
    if self.on_client_data_update:    # ← 这个回调存在
        self.on_client_data_update(cid)
```

`main.py:68-69` 只设置了两个回调：
```python
server_mgr.on_update = on_update
server_mgr.on_log = on_log
# ❌ 缺少: server_mgr.on_client_data_update = ...
```

但这只解释了 gm_tree 为空的问题。用户说"不显示设备连接"，意味着连 clients 列表都没更新到前端。

进一步分析：`on_update()` 在 `_handle_client()` 中被调用时（`server_mgr.py:134`），客户端刚连接，此时 `on_update` 确实会广播 clients 列表。但如果 WS 连接此时还没建立（前端刚加载），就会错过这个事件。

HTTP fallback 轮询（每 3 秒）应该能补救，但前提是 API 代理正常工作。如果 GM Console 子进程健康检查失败（`process_manager.check_health` 返回 false），代理会返回 503。

**修复方案**:

1. 在 `main.py` 中设置 `on_client_data_update` 回调：
```python
def on_client_data_update(client_id):
    asyncio.create_task(broadcast_event({
        "type": "update",
        "listeners": server_mgr.get_listeners_info(),
        "clients": server_mgr.get_clients_info(),
    }))

server_mgr.on_client_data_update = on_client_data_update
```

2. WS `init` 事件已包含 clients 列表，新连接的前端会立即收到当前状态 → 无需额外处理。

**涉及文件**: `tools/gm_console/main.py`

### 2.1 [P1] Hub 主页打开工具用新页签

**现象**: 点击"打开"按钮后在当前页签内导航，用户希望新页签打开。

**根因**: `Dashboard.jsx:57-59` 使用 React Router 的 `navigate()`：
```javascript
const openTool = (toolId) => {
  navigate(`/${toolId}`)  // ← 当前页签内 SPA 导航
}
```

**修复方案**: 改为 `window.open()`：
```javascript
const openTool = (toolId) => {
  window.open(`/${toolId}`, '_blank')
}
```

**涉及文件**: `frontend/src/pages/Dashboard.jsx`

### 2.2 [P1] AdbMaster Logcat 按钮 hover 效果

**现象**: Logcat 的"开始"按钮没有 hover 反馈，用户以为按钮是坏的。

**根因**: 按钮使用内联 Tailwind 类，缺少 hover 状态：
```jsx
// AdbMaster.jsx:572-576
className={`px-3 py-1.5 rounded-lg text-sm font-medium transition-all ${
  logcatRunning
    ? 'bg-[var(--terracotta)] text-white'
    : 'bg-[var(--sage)] text-white'
}`}
```

没有 `hover:` 类，鼠标悬停时无视觉变化。

**修复方案**: 添加 hover 效果（亮度变化 + 上移 + 阴影）：
```jsx
className={`px-3 py-1.5 rounded-lg text-sm font-medium transition-all cursor-pointer ${
  logcatRunning
    ? 'bg-[var(--terracotta)] text-white hover:brightness-110 hover:-translate-y-0.5 hover:shadow-md'
    : 'bg-[var(--sage)] text-white hover:brightness-110 hover:-translate-y-0.5 hover:shadow-md'
}`}
```

**涉及文件**: `frontend/src/pages/AdbMaster.jsx`

## 三、苏格拉底自检

### Q1: SPA fallback 的 catch-all 路由会不会拦截 API 请求？
**A**: 不会。FastAPI 路由按注册顺序匹配，`hub_router` 和 `proxy_router` 通过 `include_router` 先注册，优先级高于后注册的 `spa_fallback`。`/api/gm_console/clients` 会被 `proxy_router` 的 `/api/{tool_id}/{path:path}` 先匹配到。

### Q2: 为什么用户说"不显示设备连接"而不仅是"gm_tree 为空"？
**A**: 两种可能：(1) 用户打开 GM Console 页面时，游戏客户端已经连接完毕，`on_update` 的广播已经发过了，WS 的 `init` 事件应该包含 clients，但如果此时 WS 连接还没建立成功（代理延迟），就会错过；(2) HTTP fallback 轮询每 3 秒请求一次 `/api/gm_console/clients`，如果代理正常应该能拿到数据。最可能的原因是 1.1 的 404 问题导致页面根本没加载出来，用户看到的是 JSON 文本而非 React 应用。

### Q3: `window.open` 会不会被浏览器拦截为弹窗？
**A**: 不会。`window.open` 在用户点击事件的同步调用栈中执行时，浏览器不会拦截。`onClick` handler 中直接调用 `window.open` 是安全的。

### Q4: Logcat 按钮的 `brightness-110` 在 Tailwind v4 中是否可用？
**A**: Tailwind v4 支持 `brightness-*` 工具类。`brightness-110` 对应 `filter: brightness(1.1)`，是标准用法。
