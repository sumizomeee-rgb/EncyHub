## 修复范围总览

经对比原工具（HaruRuntimeGMClient、AutoSvn/FlowSVN、AdbMaster）源码与 EncyHub 当前实现，共梳理出 **4 个阶段、22 项修复任务**。按依赖关系和严重程度排序执行。

## 阶段一：基础设施修复（阻塞性问题）

### 1.1 WebSocket 代理支持 [P0 严重]

**问题**：`hub_core/api.py` 的 `proxy_router` 仅代理 HTTP，不支持 WebSocket 转发。AdbMaster Logcat 和 GmConsole 实时事件完全不可用。

**修复方案**：
- 在 `api.py` 中新增 WebSocket 代理路由 `@proxy_router.websocket("/api/{tool_id}/{path:path}")`
- 使用 FastAPI 原生 `WebSocket` + `websockets` 库建立双向转发管道
- Hub 端接受前端 WS 连接 → 向工具子进程目标端口发起 WS 连接 → 双向 relay 消息
- 连接断开时双端同步关闭

**涉及文件**：
- `hub_core/api.py` — 新增 WebSocket 代理函数

### 1.2 Toast 组件 useCallback → useMemo [P2]

**问题**：`Toast.jsx:95` 用 `useCallback` 包裹对象字面量，应为 `useMemo`。

**修复方案**：
- 将 `useCallback({...}, [addToast])` 改为 `useMemo(() => ({...}), [addToast])`

**涉及文件**：
- `frontend/src/components/Toast.jsx`

## 阶段二：全局 UI 间距与排版优化 [P1]

**问题**：所有页面间距紧凑、字号偏小，视觉拥挤。

**修复方案**：在 `index.css` 中统一调整基准样式变量和组件默认值：

| 目标 | 当前 | 调整为 |
|------|------|--------|
| `.glass-card` 内 padding | 各页面 `p-4`~`p-5` | 统一 `p-5`~`p-6` |
| 面板标题字号 | `text-sm` | `text-base` |
| 正文/描述字号 | `text-xs` 为主 | `text-sm` 为主 |
| 列表项间距 | `space-y-1.5` | `space-y-2`~`space-y-2.5` |
| 页面主内容 padding | `p-4`~`p-6` | `p-6`~`p-8` |
| 卡片间距 gap | `gap-4` | `gap-5`~`gap-6` |
| `data-table` 单元格 padding | 偏小 | 增加 `px-4 py-3` |

**涉及文件**：
- `frontend/src/index.css` — 全局基准
- `frontend/src/pages/Dashboard.jsx` — 工具描述字号
- `frontend/src/pages/AdbMaster.jsx` — 设备列表、控制中心间距

## 阶段三：功能复刻（对齐原工具）

### 3.1 GmConsole — Toggle 类型 GM 元素 [P1 高]

**问题**：原 HaruRuntimeGMClient 的 GM 树支持 `SubBox`/`Btn`/`Toggle`/`Input` 四种类型。当前 EncyHub 仅处理 `SubBox` 和默认按钮，Toggle 和 Input 被当作普通按钮。

**修复方案**：
- GM 网格渲染逻辑中，根据 `node.type` 分别渲染：
  - `SubBox` → 文件夹按钮（已有）
  - `Btn` → 普通执行按钮（已有）
  - `Toggle` → 开关组件（Switch），点击切换 true/false，调用 `exec-gm` 传递 value
  - `Input` → 输入框 + 确认按钮，输入值后调用 `exec-gm` 传递 value
- 需要维护每个客户端的 UI 状态（toggle 值、input 值），存储在组件 state 中，以 `client.id + node.id` 为 key

**涉及文件**：
- `frontend/src/pages/GmConsole.jsx` — GM 网格渲染部分

### 3.2 GmConsole — Input 类型 GM 元素 [P1 高]

与 3.1 合并实现。Input 类型渲染为：
- 左侧显示 `node.name` 标签
- 右侧为文本输入框
- 值变更或回车时调用 `handleExecGm(node.id, inputValue)`

### 3.3 GmConsole — 广播模式选择 [P2]

**问题**：原工具支持选择"全部客户端"或"指定端口"广播，当前广播固定发给所有客户端。

**修复方案**：
- 在客户端列表顶部增加一个"全部广播"虚拟选项（类似原工具的 `BROADCAST_MESH`）
- 选中时，GM 命令和 Lua 执行都走 broadcast 接口
- 选中具体客户端时走单客户端接口

**涉及文件**：
- `frontend/src/pages/GmConsole.jsx`

### 3.4 GmConsole — WebSocket 实时事件替代轮询 [P2]

**问题**：当前使用 2 秒 `setInterval` 轮询，延迟高且浪费资源。原工具通过 WebSocket 推送连接/断开/日志事件。

**修复方案**：
- 后端 `tools/gm_console/main.py` 已有 `/ws/events` WebSocket 端点
- 前端改为连接 `ws://host/api/gm_console/ws/events`（依赖阶段一 WS 代理完成）
- 收到事件时增量更新 clients、logs 等状态
- 保留一个低频轮询（10秒）作为 fallback 兜底

**涉及文件**：
- `frontend/src/pages/GmConsole.jsx`

### 3.5 AdbMaster — 设备昵称编辑 [P2]

**问题**：原 AdbMaster 支持双击/点击编辑昵称，后端已有 `PUT /devices/{hw_id}/nickname` 接口，前端未实现。

**修复方案**：
- 在控制中心设备头部区域，设备名旁增加编辑图标按钮
- 点击进入 inline 编辑模式（input 替换 span）
- Enter 确认 / Escape 取消 / blur 确认
- 调用 `PUT /api/adb_master/devices/{hw_id}/nickname` 保存
- 保存后刷新设备列表

**涉及文件**：
- `frontend/src/pages/AdbMaster.jsx`

### 3.6 AdbMaster — 路径历史记录 [P2]

**问题**：原 AdbMaster 文件传输有路径历史下拉（最近 20 条），当前无此功能。

**修复方案**：
- 需要后端支持：检查 `tools/adb_master/` 是否已有路径历史 API，若无则需新增
- 前端为推送/拉取的路径输入框增加下拉历史列表
- 选择历史项自动填充路径
- 每次成功操作后将路径加入历史

**涉及文件**：
- `tools/adb_master/main.py` — 可能需新增历史 API
- `tools/adb_master/config_manager.py` — 历史存储
- `frontend/src/pages/AdbMaster.jsx` — 历史下拉 UI

### 3.7 FlowSvn — 执行历史/日志查看 [P1 高]

**问题**：原 AutoSvn 有执行日志面板和历史记录（最近 20 次），当前任务执行后无法查看详细日志。

**修复方案**：
- 任务表格操作列增加"日志"按钮图标
- 点击弹出日志查看弹窗，显示 `task.last_log` 全文
- 增加"历史"按钮，弹窗显示 `task.execution_history` 列表（时间、状态、耗时、日志预览）
- 点击历史条目可展开查看完整日志

**涉及文件**：
- `frontend/src/pages/FlowSvn.jsx`

### 3.8 FlowSvn — 动作排序（上移/下移）[P2]

**问题**：原工具模板动作支持排序，当前无法调整顺序。

**修复方案**：
- 模板编辑弹窗中，每个动作卡片增加上移（↑）和下移（↓）按钮
- 第一个动作隐藏上移，最后一个隐藏下移
- 点击交换数组中相邻元素位置

**涉及文件**：
- `frontend/src/pages/FlowSvn.jsx` — 模板编辑弹窗

### 3.9 FlowSvn — SVN 路径验证 [P2]

**问题**：原工具验证路径是否为有效 SVN 工作副本，当前无验证。

**修复方案**：
- 后端检查：任务创建/编辑时，检查 `svn_path` 目录下是否存在 `.svn` 文件夹
- 若不存在返回明确错误提示
- 前端显示验证失败信息

**涉及文件**：
- `tools/flow_svn/main.py` — 任务创建/编辑接口增加验证

### 3.10 FlowSvn — 任务启用/禁用快捷切换 [P2]

**问题**：原工具在列表中可直接切换启用状态，当前需进入编辑弹窗。

**修复方案**：
- 任务表格中任务名称左侧的状态圆点改为可点击的开关
- 点击直接调用 `PUT /api/flow_svn/tasks/{id}` 切换 `enabled` 字段
- 切换后刷新列表

**涉及文件**：
- `frontend/src/pages/FlowSvn.jsx`
